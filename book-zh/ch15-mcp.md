# 第 15 章：MCP — 通用工具协议

## 为什么是 MCP

在 MCP（Model Context Protocol）之前，每个 agent 框架都有自己定义和发现工具的方式。一行工具是 Anthropic 特有的。另一行是 OpenAI 特有的。另一行是 LangChain 特有的。开发者必须为每个平台重写工具集成。

MCP 改变了这一点。它是工具发现和调用的标准化协议。为一个 MCP 客户端构建的工具适用于任何 MCP 客户端。Claude Code 实现了一个可在生产环境中使用的最完整的 MCP 客户端，支持 8 种传输类型和完整的 OAuth 认证流程。

## 八种传输类型

Claude Code 支持 8 种 MCP 传输类型，使 MCP 服务器在任何地方都工作：

| 传输 | 协议 | 用例 |
|----------|---------|---------|
| stdio | 子进程 stdin/stdout | 本地服务器 |
| SSE | Server-Sent Events | 远程服务器、基于 Web 的 |
| HTTP | REST 风格 | 通用远程 |
| WebSocket | 双向 | 实时服务器 |
| SDK | 进程内 | 嵌入式服务器 |
| IDE v1/v2 | IDE 桥接 | IDE 集成 |
| Claude.ai 代理 | 代理 | 远程连接 |

每种传输类型本质上做的是相同的事情：产生命名工具的列表，每个工具带有类似于内置 Claude Code 工具的输入 schema。传输类型抽象化了通信协议，因此可用的工具集对模型来说看起来相同。

> 💡 **译注**：为什么需要 8 种传输？不同部署场景需要不同的通信方式。本地开发工具用 stdio（最简单，直接启动子进程），远程 Web 服务用 SSE/HTTP（通过防火墙），IDE 集成用专用桥接，企业部署用代理。Claude Code 作为"一个产品适配所有场景"的策略，意味着它必须在传输层吃下所有这些复杂性。如果你在自己做 Agent 产品，大概率只需要 stdio 和 HTTP 两种。

## 工具包装

MCP 集成不只是暴露原始 MCP 工具。它包装它们：

- **Schema 翻译**：MCP 的 JSON Schema 被翻译为 Zod schema
- **进度报告**：MCP 进度通知成为 agent 可见的进度事件
- **错误处理**：MCP 错误被翻译为标准的 tool_result 错误
- **结果预算**：MCP 结果受制于与其他工具相同的 `maxResultSizeChars` 约束

包装器使 MCP 工具从模型的角度来看与内置工具不可区分。同样的验证，同样的权限，同样的结果限制。

## 动态工具注册

MCP 服务器可以在运行时出现和消失。服务器可以异步连接——标记为 `pending` 直到它们完全就绪。当服务器连接时，其工具被添加到集中式注册表中。当它断开连接时，其工具被移除。

工具注册表是动态的：它在每次查询轮次中反映当前的工具集，而不是启动时的快照。这意味着模型始终看到最新可用的工具，包括在会话中途连接的 MCP 服务器。

## OAuth for MCP

一些 MCP 服务器需要认证。Claude Code 实现了完整的 MCP OAuth 流程：客户端注册、授权码流程、token 刷新。这使得 MCP 能够连接到需要用户同意的第三方服务，而不共享其 API 密钥。

OAuth 流程是异步的。服务器以 pending 状态开始。认证过程在后台运行。当完成时，服务器转变为活跃状态，其工具对模型可用。

## 工具缓存和 Prompt 稳定性

MCP 工具名称和描述被包含在 prompt 缓存中。为了保持缓存稳定性（参见第 4 章），内置工具按字母顺序，MCP 工具追加在最后。这样添加或移除 MCP 服务器不会移动内置工具的位置，保持缓存前缀逐字节稳定。这与第 6 章中的 prompt-cache 断点模式相同。

## Apply This

**MCP 是通用协议。** 如果你在定义自定义工具协议，你可能在犯错误。MCP 足够好了，并且正在变得更好。为一次构建，在任何地方使用。为一个 MCP 客户端编写一次工具，它们适用于所有客户端。

**包装外部工具。** 当集成外部工具时，包装它们——不要暴露原始接口。添加进度报告、错误翻译和结果预算。使它们与内置工具不可区分。

**异步连接 MCP 服务器。** 不要在启动时阻塞。将服务器标记为 pending 并在准备好时增量添加工具。如果服务器永远不连接，用户看到一个空工具列表，而不是一个卡住的启动序列。

**OAuth 是 MCP 的超能力。** 标准化的 OAuth 支持意味着 MCP 工具可以安全连接到第三方服务而不共享 API 密钥。工具可以请求范围访问。用户可以按服务撤销访问。

**工具对模型来说都是相同的。** 内置工具、skill 工具和 MCP 工具对模型来说应该不可区分。统一接口使得容易交换 provider 或添加新工具而不改变 agent 行为。
