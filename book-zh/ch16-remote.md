# 第 16 章：远程控制与云端执行

## 超越本地主机

Claude Code 主要是一个本地 CLI。但远程执行允许 agent 在远程机器上运行同时保持本地 UI。

## Bridge 协议

Bridge 连接本地 REPL 和远程 agent loop。两个版本：

**Bridge v1**：轮询模型。本地 REPL 轮询远程端的状态变化。简单但高延迟。

**Bridge v2**：事件驱动。远程端推送事件（消息、工具调用、状态变化）到本地 REPL。较低延迟，较复杂。使用 Server-Sent Events 进行单向流和 HTTP RPC 进行命令。

## Claude Code Runtime（CCR）

CCR 是远程执行环境。它在云端运行 agent loop 同时将事件流式传输回本地客户端：
- 本地 REPL 渲染 UI 并处理用户输入
- 远程运行时执行工具、调用模型、管理上下文
- Bridge 协议同步两者

## 非对称通道

设计的核心洞察：读取和写入有不同的带宽。
- **读取**（远程 → 本地）：高频流式输出。每个 token 生成一个事件。
- **写入**（本地 → 远程）：低频 RPC。仅当用户输入或工具审批。

Bridge 为每个方向使用不同的传输。SSE 用于读取（单向、可缓存、防火墙友好）。HTTP POST 用于写入（标准、无处不在）。

## Upstream Proxy

对于企业客户，upstream 代理拦截 API 调用以用于：
- 速率限制和成本跟踪
- 审计日志
- 策略强制执行
- 自定义模型路由

代理在启动时通过环境变量和 CLI 标志配置。

## Apply This

1. **为读取和写入使用不同的通道。** 读取是高带宽流，写入是低频 RPC。为每个需求优化。
2. **SSE 用于流式事件。** 比 WebSocket 简单，通过所有代理和防火墙工作，原生自动重连。
3. **Bridge 模式分离展示和计算。** 本地渲染，远程执行。UI 保持响应，计算获得云资源。
4. **审计对企业很重要。** 如果你在构建企业 agent 基础设施，从第一天起就构建审计——不只是遥测，而是完整的请求/响应日志。
5. **Upstream 代理处理横切关注点。** 速率限制、成本跟踪、策略强制执行属于代理层，而不属于 agent loop。
