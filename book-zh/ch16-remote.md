# 第 16 章：远程控制与云端执行

## Agent 的触角伸向 Localhost 之外

Claude Code 主要是一个本地 CLI。但远程执行允许 agent 在远程机器上运行，同时保持本地 UI。这对于云开发环境、企业部署策略以及本地机器不够强大的场景是必要的。

远程架构的核心洞察：读取和写入有不同的带宽需求。读取（远程→本地）是高频率流——每个 token 生成一个事件。写入（本地→远程）是低频 RPC——仅在用户输入或工具审批时。

---

## Bridge 协议

Bridge 连接本地 REPL 和远程 agent loop。两个版本：

**Bridge v1（轮询）。** 本地 REPL 按间隔轮询远程端的状态变化。简单、可防火墙穿透、易于调试。高延迟。

**Bridge v2（事件驱动）。** 远程端通过 SSE 推送事件到本地 REPL。较低延迟，较复杂。使用 SSE 进行单向流，HTTP POST 进行命令。读取和写入使用不同的传输——SSE 适用于高带宽流式输出，HTTP POST 适用于低频命令。

协议差异对 REPL 是透明的：两种情况都看到相同的 `Message` 对象。

---

## Claude Code Runtime（CCR）

CCR 是远程执行环境。它在云端运行 agent loop，同时将事件流式传输回本地客户端。本地 REPL 渲染 UI 并处理用户输入。远程运行时执行工具、调用模型、管理上下文。Bridge 协议同步两者。

如果远程运行时配置了但连接失败，系统以警告回退到本地执行。

---

## 非对称通道

- **读取**（远程→本地）：高频率流。每个 token 一个事件。SSE——单向、可缓存、防火墙友好。
- **写入**（本地→远程）：低频 RPC。仅在用户输入或工具审批时。HTTP POST——标准、无处不在。

---

## Upstream Proxy

对于企业客户，upstream 代理拦截 API 调用用于：速率限制和成本跟踪、审计日志、策略强制执行和自定义模型路由。Agent loop 不知道 proxy——它看到的是标准的 Anthropic 客户端接口。

---

## Apply This

**为读取和写入使用不同的通道。** 读取是高带宽流，写入是低频 RPC。SSE 用于流式事件，HTTP POST 用于命令。

**Bridge 模式分离呈现和计算。** 本地渲染，远程执行。UI 保持响应，计算获得云资源。

**审计从第一天开始构建。** 不只是遥测——完整的请求/响应日志。Upstream proxy 是正确的地方。Agent loop 不应该知道审计。

**Proxy 处理横切关注点。** 速率限制、成本跟踪、策略强制执行属于 proxy 层，不属于 agent loop。
