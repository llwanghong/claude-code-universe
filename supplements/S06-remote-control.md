# S06：远程控制

> 对应：书 ch16 | 源码：bridge/, remote/

## 本地 UI，远程执行

Claude Code 支持远程执行：agent loop 在云端运行，本地终端渲染 UI。

## Bridge 协议

本地 REPL 和远程 agent loop 之间的连接层。v1 是轮询（简单，高延迟）。v2 是事件驱动（较低延迟，较复杂，使用 SSE for streaming 和 HTTP RPC for commands）。

## 非对称通道

- **读取**（远程 → 本地）：高频流。每个 token 一个事件。SSE。
- **写入**（本地 → 远程）：低频 RPC。仅在用户输入或工具审批时。HTTP POST。

## 前端启示

1. **分离渲染和计算。** 本地渲染（快，响应式），远程执行（强大的机器，大上下文窗口）
2. **为读取和写入使用不同的通道。** 读取是高带宽流，写入是低频 RPC
3. **SSE 适用于流式事件。** 简单，通过所有代理，原生重连
4. **Upstream 代理适用于横切关注点。** 速率限制、审计、策略——属于代理层，不属于 agent loop
5. **企业 AI 应用需要审计日志。** 不是遥测——AI 的每个请求/响应的完整日志
